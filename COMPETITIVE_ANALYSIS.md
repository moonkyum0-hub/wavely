# Wavely 경쟁사 심층 분석 (2026-07-19, v2)

목적: 경쟁/유사 제품을 **사용자 경험(심리) · 상업적 기능 · 편의성** 축으로 분석하고,
그 결과를 **Wavely의 계획된 기능 하나하나에 직접 매핑**해서 "무엇을 배우고, 무엇을 피하고,
어디서 차별화할지"를 결정한다.

---

## PART 1 — 제품별 프로필

### 1. Apple Health / Activity Rings + State of Mind
- **컨셉**: 3개 링(이동·운동·기립)을 매일 채우기. iOS 17+부터 State of Mind(기분 기록)와 저널 추가.
- **사용자 경험**
  - 👍 3개 링이라는 극단적 단순함 — 학습비용 제로. 시각적 완성감("링 닫기")이 강력한 훅.
  - 👍 State of Mind는 기분을 "매우 불쾌 ↔ 매우 쾌적" 슬라이더로 기록하고, **운동·수면·햇빛·마음챙김 같은 생활요인과의 상관관계를 보여줌**. UCLA 연구에서 참가자 80%가 "감정 인식이 늘었다"고 응답.
  - 👎 링/스트릭이 완벽주의·강박을 유발한다는 보고 다수 — 자정에 링 채우려고 거실을 서성이는 사용자, 스트릭 부담 때문에 워치를 아예 버리는 TikTok 트렌드까지. 섭식장애 특성과의 연관 연구(2017, Eating Behaviors)도 존재. 전문가 제안: "일일 스트릭 → 주간 일관성"으로 전환.
- **상업**: 하드웨어(워치) 판매 견인용 무료 앱. 건강데이터는 온디바이스, 본인이 선택할 때만 공유 — 프라이버시를 브랜드 자산으로 씀.
- **편의성**: 워치 없으면 반쪽. 아이폰 생태계 밖에서는 존재하지 않음.

### 2. Samsung Health (2026.6 리뉴얼)
- **컨셉**: AI 기반 **Energy Score**(0~100, 전날 활동+수면 기반 다음날 아침 제공), 5축 구조(활동/마음챙김/영양/수면/바이탈).
- **사용자 경험**
  - 👍 여러 지표를 하나의 아침 점수로 수렴 — "오늘 상태"를 한눈에.
  - 👍 Vitals: 수면 중 5개 지표(심박·HRV·호흡·피부온도·산소포화도)를 **개인 베이스라인 대비 변화**로 분석. "남과 비교"가 아니라 "어제의 나와 비교"라는 프레임.
  - 👎 리뉴얼마다 지표가 늘어남(Heart Health Score, Antioxidant Index...) — 점수 인플레이션으로 복잡도 상승. 핵심 AI 기능이 최신 워치에만 제공되어 불만.
- **상업**: 무료 앱 + 워치 판매 견인. AI 기능을 하드웨어 세대로 차등화.
- **편의성**: 갤럭시 생태계 의존. 워치 없이는 데이터 대부분 수동 입력.

### 3. Whoop / Oura Ring
- **컨셉**: 웨어러블 기반 Recovery(Whoop) / Readiness(Oura) 점수. "오늘 얼마나 밀어붙여도 되는가"를 알려주는 회복 중심 설계.
- **사용자 경험**
  - 👍 **"더 해라"가 아니라 "오늘은 쉬어도 된다"를 말해주는 거의 유일한 카테고리.** 쉼을 데이터로 정당화해줌.
  - 👍 HRV·체온·호흡 등 생리 지표를 0~100 단일 점수로 수렴, 아침에 한 번 확인.
  - 👎 Whoop은 해석에 트레이닝 지식 필요 — 일반인에게 진입장벽. 두 제품 모두 하드웨어 필수.
- **상업**: Whoop은 구독이 본체(기기 반무료). Oura는 기기 + 구독. 사용자 38%가 두 브랜드를 겹쳐 씀 — 단일 제품이 전체 니즈를 못 덮는다는 신호.
- **편의성**: Oura는 반지라 착용 마찰 최소, 수면 데이터 품질 최고 평가.

### 4. Fitbit → Google Health (2026.5 전환)
- **컨셉**: Gemini 기반 Health Coach. 구독 $9.99/월·$99.99/년 (구 $79.99에서 인상).
- **사용자 경험**: 👍 AI 코치에게 자기 데이터 기반 질문 가능("Ask Coach"). 👎 리브랜딩+가격인상 동시 진행으로 이탈 여론.
- **상업**: 무료 기본 트래킹 + AI 코칭·심층 수면분석·적응형 플랜은 유료. **Gemini AI 피드백을 프리미엄 티어로 파는 선례** — Wavely Phase 5~7 수익모델의 직접 참고 대상.
- **편의성**: 텍스트·사진·음성 멀티모달 로깅 도입 — 입력 마찰 줄이기 트렌드.

### 5. Streaks / Habitica (스트릭·게임화 습관 트래커)
- **컨셉**: Streaks는 연속일 유지가 제품의 전부. Habitica는 인생을 RPG화(습관 완료 → XP·골드, 미완료 → HP 감소, 파티원 연대 데미지).
- **사용자 경험**
  - 👍 초반 동기부여는 강력. Habitica 400만 유저.
  - 👎 **하루 놓침 → 스트릭 0 리셋 → "이번 판 망함"(abstinence violation effect) → 40일차 이탈 시작, 60일차 대부분 이탈.** 10개 습관앱 30일 테스트 후기: "8개가 같은 방식으로 나를 망가뜨렸다".
  - 👎 Habitica: "정신건강 때문에 챌린지 못 하겠다고 하니 파티에서 쫓겨났다" — 연대 압박이 사회적 처벌로 변질.
- **상업**: 일회성 구매(Streaks) / 구독+아이템(Habitica).
- **편의성**: 입력은 간단하나 유지 관리(스트릭 방어)가 심리 비용.

### 6. Finch (무처벌 자기관리)
- **컨셉**: 반려 새를 키우는 자기관리 앱. 내가 자기관리를 하면 새가 성장. **하루 놓쳐도 벌칙 없음 — 돌아오면 반갑게 맞아줄 뿐.**
- **사용자 경험**
  - 👍 "실적 관리"가 아니라 "돌봄 관계"로 프레이밍 — 자기관리가 수행(performance)이 아닌 애정 행위가 됨.
  - 👍 ADHD·불안·PTSD 사용자들이 "처음으로 계속 쓰게 된 앱"이라 증언. **Wavely 타겟(완벽주의 강박 보유자)과 가장 겹치는 성공 사례.**
  - 👎 목표지향적 사용자에겐 밋밋. 정량 데이터·분석 거의 없음.
- **상업**: 프리미엄 구독(치장 아이템 중심) — 죄책감 대신 애착으로 결제 유도.
- **편의성**: 하드웨어 불필요, 앱 단독 완결. 진입장벽 최소.

### 7. MyFitnessPal / Strava (정량 로깅)
- **사용자 경험**
  - 👎 UCL 연구(트윗 58,881건 분석): "건강하지 않은" 음식 기록 시 수치심, 알고리즘 목표(한 사용자에겐 "-700kcal 섭취" 권장이라는 오류까지)에 대한 좌절, **측정이 즐거움을 갉아먹음** — 하프마라톤 PB를 세우고도 앱이 기록 못 잡은 것에 낙담한 사례. 일부는 목표 자체를 포기 — 앱의 존재 목적과 정반대 결과.
- **교훈**: 로깅 UI의 톤과 알고리즘 목표 설정이 제품의 심리적 안전을 좌우한다.

### 8. Fitbod / Strong (근육 히트맵 — Body Map 직접 선례)
- **컨셉**: 운동 로그 → 근육군별 피로/회복도(0~100%)를 신체 그림에 색 강도로 표시. 회복 상태가 다음 워크아웃 추천에 반영.
- **사용자 경험**
  - 👍 "내가 뭘 놓치고 있는지" 시각적으로 즉시 파악 — 불균형 발견 도구로서 검증됨.
  - 👍 히트맵이 **다음 행동 추천과 연결**됨 (피로한 부위는 쉬게 하는 프로그램 생성) — 시각화가 장식이 아니라 기능.
  - 👎 여전히 "훈련량 채우기" 게이지 — 완벽주의자에게 또 하나의 채워야 할 압박이 될 수 있음.
- **상업**: 구독. 히트맵이 핵심 구독 유인.

### 9. Bearable (증상·기분 상관관계 — 상관분석 직접 선례)
- **컨셉**: 만성질환자 대상 증상·기분·수면·에너지·약물 추적 + **상관관계 리포트**. 90만 사용자.
- **사용자 경험**
  - 👍 몇 번의 탭으로 하루 기록 — 만성질환자(에너지가 부족한 사용자)를 위한 초저마찰 설계.
  - 👍 "습관/치료 변화 ↔ 증상 변화" 상관을 커스텀 그래프로 — **Wavely의 "운동이 컨디션·수행능력에 도움되는지 확인하고 싶다"는 니즈를 이미 상품화한 유일한 앱.**
  - 👍 앱스토어 4.8★ (4천 리뷰) — 이 카테고리에 진짜 수요가 있다는 증거.
- **상업**: 연 $34.99 구독 — 무제한 목표, 커스텀 실험(Experiments), **상관관계 리포트, 30일 이전 히스토리**가 유료. 상관분석을 프리미엄으로 파는 선례.
- **편의성**: 페르소나 중 "환자"를 정조준. 커스터마이징 자유도 높음.

### 10. Exist.io / Welltory (라이프스타일 상관분석)
- **컨셉**: 여러 서비스 데이터를 모아 자동 상관분석 ("여행 중엔 더 걷는다", "낮잠 안 자면 더 걷는다").
- **한계 (사용자 지적)**: **r² 값을 안 보여주고, 변수 통제도 안 됨** — "상관≠인과"라서 결국 통찰은 사용자 몫. 상관분석을 제품화할 때 통계적 정직함이 필요하다는 교훈.

### 11. Lumosity / Elevate (두뇌 훈련 — 인지검사 관련 ⚠️ 법적 선례)
- **⚠️ Lumosity는 2016년 FTC에 200만 달러 벌금.** "업무·학업 능력 향상, 치매 지연" 등을 광고했으나 과학적 근거 부족 판정. 일리노이대 연구팀의 370개 문헌 리뷰: 게임 실력이 느는 것이지 실생활 능력 전이(transfer) 증거는 거의 없음. 2025년 학계 합의도 "전이 효과는 기껏해야 논쟁적, 대체로 무효".
- **Wavely에 주는 의미**: 인지검사를 **"훈련하면 좋아진다"로 팔면 안 되고, "측정 도구"로 팔아야 한다.** Wavely의 설계(인지검사 = 운동·컨디션과의 상관을 보는 측정)는 우연히도 법적으로 안전한 프레임이지만, 마케팅 문구에서 "인지능력 향상" 류의 표현을 쓰는 순간 Lumosity와 같은 리스크가 생긴다. 정식 출시 전 문구 검수 필수.

### 12. 국내 앱: 챌린저스 / 눔 / 캐시워크
- **챌린저스** (예치금 모델): 목표에 돈을 걸고 100% 달성 시 전액+상금 환급, 실패 시 차감. 평균 달성률 90% (일반적 목표 달성률 8~9% 대비 압도적).
  - 👎 그러나: 엄격한 인증 기준(사진 규정 위반 시 무더기 미인증), 출금 최소액 장벽, 상업 콜라보 증가로 "순수한 도전 의미 퇴색" 비판.
  - **의미**: 돈을 건 손실 회피는 단기 달성률엔 최강이지만, 이건 **외적 동기 + 처벌 모델의 극단** — Wavely 철학과 정반대. 다만 90%라는 숫자는 "책임 장치(commitment device)"의 힘 자체는 증명함. Wavely는 처벌 없는 책임 장치(예: 자신과의 약속 선언, 주간 회고)로 치환할 수 있는지 고민할 가치.
- **눔**: 행동심리학 기반 다이어트 코칭, 음식 신호등 시스템. 심리학 기반 접근의 대중화 선례 — 다만 최종 목표가 체중감량이라 칼로리 제한 압박 비판도 존재.
- **캐시워크**: 걸으면 현금 리워드. 보상형 만보기의 국내 대표 — 리워드 중심이라 건강 인사이트는 얕음.

## PART 2 — Wavely 기능별 매핑

### A. 파도 점수 (단일 종합 점수) — `calcWaveScore()`
| | 내용 |
|---|---|
| 선례 | Samsung Energy Score, Whoop Recovery, Oura Readiness — 업계 전체가 "다지표 → 단일 아침 점수"로 수렴 |
| 배울 것 | Samsung의 "개인 베이스라인 대비 변화" 프레임(남과 비교 X, 어제의 나와 비교) — Wavely의 델타 인디케이터와 정확히 일치. Whoop/Oura의 "오늘은 쉬어도 된다" 방향의 피드백 |
| 피할 것 | Samsung의 점수 인플레이션(Energy + Heart + Antioxidant...) — 지표가 늘어도 **홈 화면은 파도 점수 하나** 원칙 유지. 세분화 지표(신진대사, 뇌혈류 등 확장 시)는 상세 화면으로 |
| 차별화 | 경쟁사 점수는 "오늘의 준비도"만 말함. 파도 점수는 **낮은 날도 흐름의 일부**라는 서사(파도)로 감싸는 유일한 점수 |

### B. 스트릭 / 연속 기록 — Dashboard 연속기록 카드
| | 내용 |
|---|---|
| 선례(반면교사) | Streaks·Habitica·Apple Rings — 리셋형 스트릭의 실패는 연구·후기로 입증 (40~60일차 대량 이탈) |
| 배울 것 | Finch: 놓친 날에 벌칙 대신 "돌아온 것을 환영" — 복귀 UX가 핵심. 전문가 제안 "주간 일관성" 프레임 |
| 적용 | ① 연속기록이 끊겨도 0으로 리셋하는 시각 표현 금지 — Fade(안개)로 흐려질 뿐. ② "이번 주 5/7일 기록" 같은 주간 밀도 표현을 스트릭보다 상위에. ③ 공백 후 복귀 시 전용 환영 메시지("파도는 다시 밀려오죠" 류) — 복귀일을 특별하게 |

### C. Body Map (신체 지도)
| | 내용 |
|---|---|
| 선례 | Fitbod 근육 히트맵 — 부위별 색 강도 + 회복도 %, 다음 워크아웃 추천에 연결 |
| 배울 것 | ① 시각화가 장식이 아니라 **다음 행동과 연결**되어야 함 (Fitbod: 피로 부위 → 휴식 추천). ② 불균형 발견 도구로서의 가치 |
| 피할 것 | "채워야 할 게이지" 프레임 — 커버리지 %, 부위별 점수 랭킹 금지 (기존 결정과 일치) |
| 차별화 | Fitbod은 근육·피로만 다룸. Wavely는 **신진대사·뇌혈류·인지까지 "몸의 기능"을 세분화**해서 보여주는 것이 목표 — 이 확장이 유일무이한 지점. 단, 기능별 수치는 실측 불가능한 게 많으므로(뇌혈류 등) "측정값"이 아니라 **"이 활동이 이 기능에 기여한다고 알려져 있어요"라는 지식 기반 피드백**으로 프레이밍해야 정직함(데이터 정직성 원칙과 Lumosity 교훈 모두 충족) |
| 디자인 숙제 | 사용자 피드백: 현 마네킹 시안은 ① 테마와 이질적 ② 3D 필요 ③ 세분화 부족. Fitbod도 2D 평면 그림자 스타일이므로, 3D + 기능 세분화(근육 외 장기·혈류·뇌)로 가면 카테고리에서 시각적으로 앞설 수 있음 |

## PART 3 — 요약: Wavely의 포지션

**시장의 빈자리**: 현재 시장은 (a) 성과 압박형(Apple, Whoop, MyFitnessPal, 챌린저스), (b) 무처벌 정서형(Finch — 단 데이터 없음), (c) 상관분석형(Bearable — 단 질환 관리 방향)으로 갈라져 있고, **"무처벌 정서 + 정량 데이터 + 성장 방향 상관분석"을 동시에 하는 제품은 없다.** Wavely가 노리는 자리가 정확히 이 교집합.

**한 줄 포지셔닝**: "Finch의 마음으로, Whoop의 데이터를, Bearable의 방법론으로 — 완벽주의자가 지치지 않고 성장을 확인하는 앱."

**최우선 실행 항목** (이 분석에서 도출):
1. Body Map을 "기능 세분화 + 지식 기반 피드백" 프레임으로 재설계 (C)
2. 인지검사·기능 피드백의 모든 카피에서 "향상 주장" 금지 — 측정·관찰 언어로 (E, Lumosity 교훈)
3. 복귀 UX(공백 후 돌아온 날의 환영)를 Phase 3 범위에 추가 검토 (B, Finch 교훈)
4. 상관분석 UI에 정직성 장치(표본 수, "경향" 언어) 설계 반영 (D, Exist.io 교훈)
